# Token Reduction for Initial Interactions

I have implemented token optimizations that target the 18k token overhead for "hi" messages and new sessions.

## Changes Made

### 1. Modular System Prompt
In `src/agents/system-prompt.ts`, I introduced a `first-turn` mode and an `excludeSections` filter.
- Optimized sections to be more concise (e.g., merging Safety lines).
- Implemented logic to skip non-essential sections like CLI references, messaging guides, and documentation when in `first-turn` mode.

### 2. Intelligent Tool Pruning
In `src/agents/pi-tools.ts`, I implemented a tool pruning mechanism.
- Created a `FIRST_TURN_TOOL_ALLOWLIST` containing only essential tools (`read`, `ls`, `grep`, `find`, `message`, `session_status`, etc.).
- Complex tools mapping large schemas (like `browser`, `canvas`, `exec`, `apply_patch`) are omitted during the first turn.

### 3. Automatic First-Turn Detection
In `src/agents/pi-embedded-runner/run/attempt.ts`, I added logic to detect if it's a new session (no history file).
- The runner now automatically applies `first-turn` prompt mode and tool pruning for fresh sessions.

## Verification Results

### Automated Tests
I ran the following tests to ensure functional correctness:
- **System Prompt Tests**: `pnpm test src/agents/system-prompt.test.ts` - **PASSED**
- **Tool Logic Tests**: `pnpm test src/agents/pi-tools.test.ts` - **PASSED**

### Estimated Token Savings
- **System Prompt**: ~8,000 tokens -> **~2,500 tokens** (68% reduction)
- **Tool Definitions**: ~10,000 tokens -> **~3,000 tokens** (70% reduction)
- **Total Initial Input**: ~18,000 tokens -> **~5,500 tokens**

> [!NOTE]
> For subsequent turns, the full toolset and prompt are restored to ensure the agent has its full capabilities for task execution. The initial "hi" or greeting is now significantly faster and cheaper.
